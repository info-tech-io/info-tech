# Этап 2: Техническая инфраструктура и шаблоны

## Цель этапа
Подготовить техническую основу для всех типов сайтов: расширить hugo-templates новыми шаблонами, создать стандартизированную структуру docs/ и настроить базовые CI/CD workflows.

## Задачи

### 2.1 Расширение hugo-templates новыми шаблонами (2-3 дня)
**Цель:** Добавить поддержку корпоративных сайтов и документации

**Действия:**
- Исследовать и выбрать подходящие Hugo темы для корпоративного сайта
- Создать новый шаблон `corporate` в hugo-templates/templates/
- Создать новый шаблон `documentation` в hugo-templates/templates/
- Настроить компоненты для блога, новостей, каталога продуктов
- Адаптировать Quiz Engine для интеграции с новыми шаблонами

**Технические требования:**
```
hugo-templates/
├── templates/
│   ├── educational/     # Существующий шаблон для модулей ИНФОТЕКА
│   ├── corporate/       # Новый: для сайта организации
│   └── documentation/   # Новый: для документации продуктов
├── themes/
│   ├── compose/         # Существующая тема
│   ├── [corporate-theme]/ # Новая тема для корпоративного сайта
│   └── [docs-theme]/    # Новая тема для документации
```

**Выходные артефакты:**
- Функциональный шаблон `corporate` с примером сборки
- Функциональный шаблон `documentation` с примером сборки
- Обновленные схемы JSON Schema для валидации module.json
- Документация по использованию новых шаблонов

### 2.2 Стандартизация структуры docs/ (1 день)
**Цель:** Создать единообразную структуру документации во всех продуктах

**Действия:**
- Создать стандартную структуру docs/ с примером module.json
- Создать шаблонные файлы документации (README, Architecture, API Reference)
- Разработать guidelines для ведения документации
- Создать автоматизацию для инициализации docs/ в новых проектах

**Стандартная структура:**
```
[product]/docs/
├── module.json          # Конфигурация сборки документации
├── content/
│   ├── _index.md       # Главная страница продукта
│   ├── getting-started/ # Быстрый старт
│   ├── architecture/   # Архитектура
│   ├── api/           # API документация
│   └── contributing/  # Гайды для контрибьюторов
└── .github/workflows/
    └── build-docs.yml  # CI/CD для документации
```

**Выходные артефакты:**
- Шаблон структуры docs/
- Примеры module.json для документации
- Guidelines по документированию
- CLI команда для инициализации docs/

### 2.3 Базовые CI/CD workflows (2 дня)
**Цель:** Настроить автоматическую сборку и синхронизацию всех типов сайтов

**Действия:**
- Создать workflow для сборки корпоративного сайта
- Создать workflow для сборки документации продуктов
- Настроить GitHub Pages для тестового деплоя
- Создать workflow для синхронизации с production сервером
- Настроить автоматические уведомления между репозиториями

**Workflows для создания:**
1. **build-corporate-site.yml** (в info-tech): Сборка главного сайта
2. **build-product-docs.yml** (шаблон): Сборка документации продукта
3. **sync-to-production.yml**: Синхронизация с production сервером
4. **notify-hub.yml** (в продуктах): Уведомление о изменениях в docs/

**Выходные артефакты:**
- Функциональные GitHub Actions workflows
- Тестовые сайты на GitHub Pages
- Документация по CI/CD процессам
- Схемы взаимодействия между репозиториями

### 2.4 Настройка GitHub Pages для тестирования (0.5 дня)
**Цель:** Подготовить тестовую среду для проверки сайтов

**Действия:**
- Настроить GitHub Pages для тестового деплоя корпоративного сайта
- Настроить GitHub Pages для тестовых сайтов документации
- Создать тестовые URL структуры для проверки
- Настроить автоматический деплой при изменениях

**Тестовые URL:**
- `info-tech-io.github.io/info-tech` - тестовый корпоративный сайт
- `info-tech-io.github.io/quiz-docs` - тестовая документация Quiz Engine
- `info-tech-io.github.io/hugo-templates-docs` - тестовая документация Hugo Templates

## Контрольные процедуры

### Критерии завершения этапа
- [ ] hugo-templates содержит функциональные шаблоны `corporate` и `documentation`
- [ ] Все продукты имеют стандартизированную структуру docs/
- [ ] CI/CD workflows работают автоматически
- [ ] Тестовые сайты доступны на GitHub Pages
- [ ] JSON Schema валидация работает для всех типов module.json
- [ ] Документация по техническим процессам создана

### Технические проверки
1. **Функциональность шаблонов:**
   - Корректная сборка сайтов hugo-templates с новыми шаблонами
   - Правильная работа компонентов (блог, навигация, поиск)
   - Адаптивность и производительность

2. **CI/CD процессы:**
   - Автоматическая сборка при изменении контента
   - Корректная работа уведомлений между репозиториями
   - Успешный деплой на GitHub Pages

3. **Стандартизация:**
   - Единообразие структуры docs/ во всех продуктах
   - Валидация module.json через JSON Schema
   - Совместимость с существующими процессами ИНФОТЕКА

### Тестирование производительности
- Время сборки сайтов < 2 минут
- Время загрузки страниц < 2 секунд
- Корректная работа на мобильных устройствах

## Риски и митигация

### Возможные риски
1. **Сложность интеграции:** Новые шаблоны конфликтуют с существующими
   - *Митигация:* Поэтапное тестирование, изоляция изменений

2. **Производительность CI/CD:** Слишком медленные процессы сборки
   - *Митигация:* Оптимизация workflows, кэширование зависимостей

3. **Совместимость тем:** Выбранные Hugo темы не подходят для наших целей
   - *Митигация:* Предварительное тестирование нескольких вариантов

### Технические ограничения
- Совместимость с текущей версией Hugo
- Ограничения GitHub Pages по объему и трафику
- Ресурсы VPS для production деплоя
