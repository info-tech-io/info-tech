# Этап 4: Деплой и инфраструктурная подготовка

## Цель этапа
Запустить все сайты в production, настроить собственную инфраструктуру на info-tech.io, обеспечить автоматическую синхронизацию контента и подготовить систему мониторинга.

## Задачи

### 4.1 Подготовка production инфраструктуры (2 дня)
**Цель:** Настроить Apache2 для обслуживания множественных доменов и поддоменов

**Технические требования:**
- Основной домен: `info-tech.io`
- Поддомены: `quiz.info-tech.io`, `hugo.info-tech.io`, `terminal.info-tech.io`
- SSL сертификаты для всех доменов
- Настройка DNS записей

**Действия:**
1. **DNS конфигурация:**
   ```
   info-tech.io              → A → [IP VPS]
   www.info-tech.io          → A → [IP VPS]
   *.info-tech.io            → A → [IP VPS] (wildcard)
   quiz.info-tech.io         → CNAME → info-tech.io
   hugo.info-tech.io         → CNAME → info-tech.io
   terminal.info-tech.io     → CNAME → info-tech.io
   ```

2. **Apache2 конфигурация:**
   ```apache
   <VirtualHost *:443>
       ServerName info-tech.io
       ServerAlias *.info-tech.io
       DocumentRoot /var/www/info-tech.io

       # Роутинг поддоменов
       RewriteEngine On
       RewriteCond %{HTTP_HOST} ^([^.]+)\.info-tech\.io$
       RewriteRule ^(.*)$ /%1/$1 [L]

       # SSL Configuration
       SSLEngine on
       SSLCertificateFile /etc/letsencrypt/live/info-tech.io/fullchain.pem
       SSLCertificateKeyFile /etc/letsencrypt/live/info-tech.io/privkey.pem
   </VirtualHost>
   ```

3. **Файловая структура:**
   ```
   /var/www/info-tech.io/
   ├── index.html              # Главный сайт
   ├── quiz/                   # Документация Quiz Engine
   ├── hugo/                   # Документация Hugo Templates
   ├── terminal/               # Документация Web Terminal
   └── assets/                 # Общие ресурсы
   ```

**Выходные артефакты:**
- Настроенный Apache2 с поддержкой всех доменов
- SSL сертификаты для всех поддоменов
- Корректная DNS конфигурация

### 4.2 Настройка SSL сертификатов (0.5 дня)
**Цель:** Обеспечить безопасное HTTPS соединение для всех сайтов

**Действия:**
- Получить wildcard SSL сертификат для *.info-tech.io
- Настроить автоматическое обновление сертификатов
- Конфигурировать security headers
- Настроить принудительное перенаправление HTTP → HTTPS

**Security headers для настройки:**
```apache
Header always set Strict-Transport-Security "max-age=31536000; includeSubDomains"
Header always set X-Frame-Options "SAMEORIGIN"
Header always set X-Content-Type-Options "nosniff"
Header always set Referrer-Policy "strict-origin-when-cross-origin"
```

**Проверки безопасности:**
- SSL Labs Grade A+ для всех доменов
- Корректная работа HSTS
- Отсутствие уязвимостей в конфигурации

### 4.4 Настройка автоматической синхронизации (1 день)
**Цель:** Обеспечить автоматическое обновление production сайтов при изменении контента

**CI/CD Workflows для настройки:**

1. **Corporate site workflow (info-tech репозиторий):**
   ```yaml
   name: Build and Deploy Corporate Site
   on:
     push:
       paths: ['docs/**']
   jobs:
     build-and-deploy:
       - Build site with hugo-templates
       - Deploy to GitHub Pages (staging)
       - Deploy to production server
   ```

2. **Product docs workflow (каждый продукт):**
   ```yaml
   name: Build and Deploy Product Documentation
   on:
     push:
       paths: ['docs/**']
   jobs:
     build-and-deploy:
       - Build docs with hugo-templates
       - Deploy to GitHub Pages (staging)
       - Deploy to production subdomain
   ```

3. **Production sync workflow:**
   - Синхронизация изменений с production сервером
   - Автоматический reload Apache при необходимости
   - Уведомления о статусе деплоя

**Мониторинг и уведомления:**
- Slack/Email уведомления о деплоях
- Мониторинг доступности сайтов
- Логирование ошибок деплоя

## Риски и митигация

### Возможные риски
1. **DNS проблемы:** Неправильная настройка DNS записей
   - *Митигация:* Тщательная проверка DNS конфигурации, тестирование с разных локаций

2. **SSL проблемы:** Проблемы с wildcard сертификатами
   - *Митигация:* Использование проверенного процесса Let's Encrypt, резервные планы

3. **Производительность:** Медленная загрузка при множественных сайтах
   - *Митигация:* Оптимизация Apache конфигурации, мониторинг ресурсов сервера

4. **Доступность:** Downtime во время переключения
   - *Митигация:* Постепенный переход, тестирование в нерабочее время

### Ресурсные ограничения
- Производительность VPS при множественных сайтах
- Ограничения Let's Encrypt на количество сертификатов
- Сетевые ресурсы для синхронизации контента
