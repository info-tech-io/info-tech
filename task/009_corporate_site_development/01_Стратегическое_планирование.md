# Этап 1: Стратегическое планирование и информационная архитектура

## Цель этапа
Детализировать контентную стратегию, адаптировать существующую концепцию из `info-tech/site/` под новую архитектуру и создать четкую информационную структуру для всех сайтов экосистемы.

## Задачи

### 1.1 Анализ и адаптация существующей стратегии (1 день)
**Исходные материалы:** `info-tech/site/00_README.md`, `02_CONTENT_STRUCTURE.md`, `03_PAGE_STRUCTURE.md`

**Действия:**
- Проанализировать существующую стратегию сайта из папки `site/`
- Адаптировать модели данных Headless CMS под Markdown + YAML frontmatter
- Пересмотреть структуру страниц с учетом новых целей (фокус на разработчиков)
- Определить различия между корпоративным сайтом и сайтами документации продуктов

**Выходные документы:**
- `01_Стратегия_корпоративного_сайта.md`
- `02_Модели_данных_Markdown.md`

### 1.2 Информационная архитектура корпоративного сайта (1 день)
**Цель:** Детализировать структуру главного сайта info-tech.io

**Действия:**
- Определить навигационную структуру сайта
- Детализировать контент каждой страницы
- Создать карту пользовательских путей (user journeys) для разных типов посетителей
- Определить приоритеты контента по этапам разработки

**Выходные документы:**
- `03_Информационная_архитектура.md`
- `04_User_Journeys.md`

### 1.3 Стандартизация документации продуктов (1 день)
**Цель:** Определить единую структуру docs/ для всех продуктов

**Действия:**
- Проанализировать текущее состояние документации в репозиториях quiz, hugo-templates, web-terminal
- Создать стандартную структуру папки docs/ с module.json
- Определить шаблоны контента для разных типов документации
- Создать guidelines для документирования продуктов

**Выходные документы:**
- `05_Стандарт_документации.md`
- `06_Структура_docs.md`

### 1.4 Контентная стратегия и план миграции (0.5 дня)
**Цель:** Определить что, откуда и куда мигрировать

**Действия:**
- Составить инвентарь существующего контента в репозитории info-tech
- Определить контент для создания с нуля
- Создать план миграции существующих markdown файлов
- Определить приоритеты создания нового контента

**Выходные документы:**
- `07_План_миграции_контента.md`
- `08_Контентная_стратегия.md`

## Контрольные процедуры

### Критерии завершения этапа
- [ ] Все 8 выходных документов созданы и проверены
- [ ] Стратегия адаптирована под новые цели (фокус на разработчиков)
- [ ] Определена четкая структура docs/ для всех продуктов
- [ ] План миграции контента детализирован с приоритетами
- [ ] User journeys проработаны для всех целевых аудиторий

### Проверочные вопросы
1. **Ясность целей:** Понятно ли из документов, как сайт будет решать задачу привлечения контрибьюторов?
2. **Техническая совместимость:** Совместимы ли предлагаемые решения с существующей архитектурой hugo-templates?
3. **Масштабируемость:** Готова ли структура к добавлению новых продуктов в будущем?
4. **Полнота планирования:** Учтены ли все необходимые типы контента и страниц?

### Валидация с заинтересованными сторонами
- Проверка соответствия целям организации
- Валидация технических решений на совместимость с существующей инфраструктурой
- Проверка user journeys на реалистичность и полноту

## Риски и митигация

### Возможные риски
1. **Переоценка сложности:** Слишком сложная структура сайта для начального запуска
   - *Митигация:* Определить MVP версию сайта и план поэтапного развития

2. **Несовместимость с hugo-templates:** Требования не вписываются в текущую архитектуру
   - *Митигация:* Постоянная проверка технической реализуемости

3. **Недооценка объема контента:** Слишком много контента для создания/миграции
   - *Митигация:* Четкая приоритизация и возможность запуска с минимальным контентом
