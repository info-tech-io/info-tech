# Задача 009: Разработка корпоративного сайта организации info-tech.io

## Цель задачи

Создать публичный сайт организации info-tech.io для привлечения сообщества разработчиков, позиционирования проектов и организации эффективного онбординга контрибьюторов.

## Контекст и обоснование

### Стратегическая необходимость
- **Разделение брендов:** Отделить бренд организации (`info-tech.io`) от продуктового бренда (`infotecha.ru`)
- **Привлечение контрибьюторов:** Создать центральную точку для онбординга разработчиков
- **Масштабируемость:** Подготовить инфраструктуру для будущих продуктов организации
- **Профессиональный имидж:** Повысить доверие партнеров и сообщества

### Целевые аудитории
1. **Разработчики-контрибьюторы** (приоритет #1) - onboarding, техническая архитектура, contributing guidelines
2. **Потенциальные партнеры и коллабораторы** - имидж, миссия, достижения
3. **Студенты образовательных программ** - обзор продуктов и экосистемы

## Архитектурный подход

### Техническая стратегия
- **Единая экосистема:** Масштабирование проверенной модели ИНФОТЕКА (Hugo + hugo-templates + GitHub Actions)
- **Source of Truth:** Markdown в репозиториях как единственный источник правды
- **Автоматизация:** Полная синхронизация контента через CI/CD workflows
- **Поэтапная миграция:** GitHub Pages → собственная инфраструктура

### Доменная архитектура
```
info-tech.io                     # Главный сайт организации
├── quiz.info-tech.io           # Документация Quiz Engine
├── hugo.info-tech.io           # Документация Hugo Templates
├── terminal.info-tech.io       # Документация Web Terminal
└── [future-products].info-tech.io
```

### Источники контента
```
info-tech/docs/                 # → info-tech.io
quiz/docs/                      # → quiz.info-tech.io
hugo-templates/docs/            # → hugo.info-tech.io
web-terminal/docs/              # → terminal.info-tech.io
```

## Разбивка на этапы

### Этап 1: Стратегическое планирование и информационная архитектура
**Срок:** 3-4 дня
**Цель:** Детализировать контентную стратегию и структуру информации

### Этап 2: Техническая инфраструктура и шаблоны
**Срок:** 5-7 дней
**Цель:** Подготовить техническую основу для всех типов сайтов

### Этап 3: Контентное наполнение и миграция
**Срок:** 7-10 дней
**Цель:** Создать и структурировать весь необходимый контент

### Этап 4: Деплой и инфраструктурная подготовка
**Срок:** 4-5 дней
**Цель:** Запустить сайты и подготовить production инфраструктуру
