### 1.3. Создание системы JSON Schema (1 день)

**Задачи**:
- Создать collection.json - главный файл схемы генераторов
- Определить базовые схемы в директории schemas/
- Настроить валидацию параметров командной строки
- Подготовить схемы для IDE автодополнения

**Файлы схем**:
- `collection.json` - описание доступных команд
- `schemas/build.json` - схема команды build
- `schemas/template.json` - схема описания шаблонов
- `schemas/component.json` - схема описания компонентов

**Пример collection.json**:
```json
{
  "$schema": "./schemas/collection.json",
  "schematics": {
    "build": {
      "description": "Собрать сайт Hugo с указанными параметрами",
      "factory": "./bin/build-factory",
      "schema": "./schemas/build.json"
    }
  }
}
```

**Критерии приемки**:
- [ ] collection.json создан и валиден
- [ ] Базовые схемы определены и работают
- [ ] Валидация параметров функционирует
- [ ] IDE автодополнение настроено

## Способы проверки

### Проверка структуры репозитория
```bash
# Проверить создание репозитория
gh repo view info-tech-io/hugo-templates

# Проверить структуру директорий
tree hugo-templates/ -L 2
```

### Проверка npm конфигурации
```bash
# Проверить валидность package.json
npm config list
npm pack --dry-run

# Проверить корректность bin скрипта
npm link
which hugo-templates
```

### Проверка JSON Schema
```bash
# Валидация схем
jsonschema -i collection.json schemas/collection-schema.json
jsonschema -i schemas/build.json schemas/build-meta-schema.json
```

### Проверка командной строки
```bash
# Основные команды
./bin/hugo-templates.js --help
./bin/hugo-templates.js build --help
./bin/hugo-templates.js list templates

# Валидация параметров
./bin/hugo-templates.js build --template=nonexistent # должно выдать ошибку
```

### Проверка default template
```bash
# Сравнение с hugo-base
cd templates/default/
hugo config
ls -la static/ themes/ content/

# Тестовая сборка
hugo --minify --destination ../../test-output/
```

## Риски этапа и их снижение

### Высокие риски

**Сложность настройки JSON Schema**
- Риск: Валидация может оказаться слишком сложной для быстрой реализации
- Снижение: Начать с минимальных схем, постепенно расширять функциональность
- Индикаторы: Время настройки превышает 1 день

**Потеря функциональности при копировании hugo-base**
- Риск: При переносе могут потеряться критические компоненты
- Снижение: Тщательное сравнение до и после, создание чек-листа компонентов
- Индикаторы: Тесты сборки не проходят, отсутствуют ключевые файлы

### Средние риски

**Проблемы совместимости npm пакета**
- Риск: package.json может иметь некорректные зависимости
- Снижение: Тестирование на чистом окружении, использование npm ci
- Индикаторы: Ошибки при npm install, проблемы с bin скриптами
