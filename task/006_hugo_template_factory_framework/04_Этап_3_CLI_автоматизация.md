# Этап 3: Командная строка и автоматизация

**Продолжительность**: 5 дней
**Цель**: Создать полноценный инструмент командной строки с автоматизацией

## Подэтапы реализации

### 3.1. Разработка главного скрипта build.sh (2 дня)
- Полная параметризация CLI (--template, --theme, --components, --content, --output)
- Система валидации входных параметров
- Интеграция с components.yml парсингом
- Логирование процесса сборки
- Обработка ошибок и понятные сообщения

### 3.2. Node.js обертка для npm интеграции (1 день)
- Создание scripts/factory.js как обертки для build.sh
- Интеграция с JSON Schema валидацией
- Настройка npm bin entry point
- Тестирование глобальной установки

### 3.3. Система валидации с JSON Schema (1 день)
- Реализация scripts/validate.js
- Валидация комбинаций template+theme+components
- Проверка существования файлов и зависимостей
- Интеграция с CLI для предотвращения ошибок

### 3.4. Вспомогательные утилиты (0.5 дня)
- hugo-templates list templates/themes/components
- hugo-templates validate --config=path
- hugo-templates generate --help
- Система автодополнения bash/zsh

### 3.5. Система логирования и отладки (0.5 дня)
- Уровни логирования (verbose, debug, quiet)
- Цветной вывод для лучшей читаемости
- Progress indicators для длительных операций
- Diagnostic режим для troubleshooting

## Критерии завершения этапа
- [ ] CLI принимает все запланированные параметры
- [ ] Валидация предотвращает некорректные комбинации
- [ ] Error messages понятны и информативны
- [ ] Help система исчерпывающе описывает возможности
- [ ] npm package работает глобально после установки
- [ ] Логирование позволяет отладить процесс сборки
- [ ] Производительность приемлема для практического использования
