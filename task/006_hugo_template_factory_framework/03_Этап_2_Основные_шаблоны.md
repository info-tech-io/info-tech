- Создать валидацию структуры components.yml
- Подготовить examples для различных конфигураций

**Структура components.yml**:
```yaml
schema_version: "1.0"
components:
  quiz-engine:
    version: "^1.0.0"
    status: "stable"
    static_files: ["quiz/", "js/quiz.js"]
    description: "Интерактивная система тестирования"
  navigation:
    version: "^1.0.0"
    status: "stable"
    static_files: ["js/nav.js", "css/nav.css"]
    description: "Базовая система навигации"
  analytics:
    version: "^0.1.0"
    status: "planned"
    description: "Система аналитики (планируется)"
```

## Способы проверки

### Проверка функциональности шаблонов
```bash
# Тестирование default template
cd templates/default/
hugo --minify --destination ../../test-output/default/
ls -la ../../test-output/default/

# Тестирование minimal template
cd ../minimal/
time hugo --minify --destination ../../test-output/minimal/
du -sh ../../test-output/minimal/
```

### Проверка components.yml
```bash
# Валидация components.yml файлов
find templates/ -name "components.yml" -exec yamllint {} \;

# Парсинг components.yml
node scripts/parse-components.js templates/default/components.yml
```

### Проверка Git submodules
```bash
# Статус submodules
git submodule status

# Инициализация submodules
git submodule update --init --recursive

# Проверка интеграции Quiz Engine
ls -la components/quiz-engine/
ls -la themes/compose/
```

### Тестирование с реальными модулями
```bash
# Клонирование тестового модуля
git clone ../mod_linux_base test-content/

# Сборка с default template
./scripts/build.sh --template=default --theme=compose --content=test-content/ --output=test-default/

# Сборка с minimal template
time ./scripts/build.sh --template=minimal --theme=compose --content=test-content/ --output=test-minimal/

# Сравнение результатов
diff -r test-default/ test-minimal/ | head -20
```

## Риски этапа и их снижение

### Высокие риски

**Потеря совместимости с hugo-base**
- Риск: Default template может потерять критическую функциональность
- Снижение: Детальное сравнение до и после, автоматизированные тесты
- Индикаторы: Ошибки при сборке существующих модулей

**Недостаточная оптимизация minimal template**
- Риск: Minimal template может не показать значительного улучшения производительности
- Снижение: Агрессивная оптимизация, измерение метрик на каждом шаге
- Индикаторы: Время сборки улучшено менее чем на 50%

### Средние риски

**Сложность Git submodules**
- Риск: Пользователи могут столкнуться со сложностями в работе с submodules
- Снижение: Автоматизированные скрипты инициализации, подробная документация
- Индикаторы: Ошибки при клонировании репозитория

**Переусложнение components.yml**
- Риск: Система компонентов может стать слишком сложной
- Снижение: Простые схемы, примеры использования, валидация
- Индикаторы: Сложности в создании новых components.yml файлов
