# Этап 4: Всестороннее тестирование

**Продолжительность**: 4 дня
**Цель**: Создать полную систему тестирования для всех компонентов

## Подэтапы реализации

### 4.1. Модульные тесты (1 день)
- Unit тесты для отдельных templates
- Тестирование парсинга components.yml
- Тестирование CLI аргументов и валидации
- Тестирование utilities и helper функций

### 4.2. Интеграционные тесты (1.5 дня)
- Тесты сборки с реальными модулями Hugo
- Тестирование всех комбинаций template+theme+components
- End-to-end тесты CLI workflow
- Тестирование с различными типами контента

### 4.3. Бенчмаркинг производительности (1 день)
- Сравнение времени сборки minimal vs default vs hugo-base
- Измерение размера выходных файлов
- Тестирование времени загрузки страниц
- Профилирование узких мест производительности

### 4.4. CI/CD настройка (0.5 дня)
- GitHub Actions workflows для автотестирования
- Тестирование на разных версиях Node.js и Hugo
- Automated testing при каждом commit/PR
- Coverage reporting и качественные метрики

## Критерии завершения этапа
- [ ] Все критические пути покрыты тестами (>80% coverage)
- [ ] Производительность не хуже hugo-base baseline
- [ ] CI/CD автоматически запускается при изменениях
- [ ] Интеграционные тесты проходят с существующими модулями
- [ ] Performance benchmarks задокументированы
- [ ] Все тесты проходят на поддерживаемых версиях Hugo
