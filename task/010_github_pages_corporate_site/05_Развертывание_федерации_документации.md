- Добавить ссылки обратно на корпоративный сайт

- Настроить SEO метаданные для всех страниц документации

### 5.2 Настройка автоматической сборки документации (1 день)
**Цель:** Обеспечить автоматическое обновление документации при изменениях

**Техническая схема для каждого продукта:**
```
Изменение в {product}/docs/content/
    ↓ (GitHub push event)
{product}/.github/workflows/notify-hub.yml
    ↓ (repository_dispatch event: "docs-{product}-update")
info-tech-io.github.io/.github/workflows/deploy-complete-sites.yml
    ↓ (Hugo build с documentation шаблоном)
GitHub Pages: info-tech-io.github.io/docs/{product}/
```

**Действия:**
- Проверить и обновить notify-hub.yml во всех продуктовых репозиториях
- Обновить deploy-complete-sites.yml для обработки документации
- Настроить правильные BaseURL для каждого документационного сайта
- Протестировать автоматическую сборку для каждого продукта
- Настроить параллельную сборку нескольких сайтов

**Конфигурация для каждого продукта:**
- Источник: `{product}/docs/` → Результат: `info-tech-io.github.io/docs/{product}/`
- Шаблон: `documentation` из hugo-templates
- Тема: `compose` (адаптированная для документации)
- BaseURL: `https://info-tech-io.github.io/docs/{product}/`

### 5.3 Интеграция навигации между всеми сайтами (1 день)
**Цель:** Создать единую навигационную систему для всей федерации

**Навигационная архитектура:**
```
Главный сайт (/)
├── Header navigation:
│   ├── Home → /
│   ├── Products → /products/ (со ссылками на документацию)
│   ├── Open Source → /open-source/
│   └── Blog → /blog/
│
Документационные сайты (/docs/{product}/)
├── Header navigation:
│   ├── ← Back to info-tech.io (ссылка на главный сайт)
│   ├── Product Home → /docs/{product}/

│   ├── User Guide → /docs/{product}/user-guide/
│   ├── Developer → /docs/{product}/developer/
│   └── Contributing → /docs/{product}/contributing/
│
Footer (одинаковый для всех):
├── Quick Links:
│   ├── Quiz Engine Docs → /docs/quiz/
│   ├── Hugo Templates Docs → /docs/hugo-templates/
│   ├── Web Terminal Docs → /docs/web-terminal/
│   └── CLI Docs → /docs/info-tech-cli/
└── Organization:
    ├── GitHub Organization → https://github.com/info-tech-io
    ├── Contributing → /open-source/contributing/
    └── Contact → mailto:info@info-tech.io
```

**Действия:**
- Создать единые компоненты header и footer для всех сайтов
- Настроить breadcrumbs навигацию в документации
- Добавить "Edit this page" ссылки на каждую страницу документации
- Создать search функциональность (если возможно с Hugo)
- Настроить правильные canonical URLs для SEO

## Риски и митигация

### Возможные риски
1. **Конфликты сборки:** Параллельная сборка нескольких сайтов вызывает ошибки
   - *Митигация:* Очередность сборки, изоляция процессов, тестирование

2. **Broken links:** Неработающие ссылки между сайтами из-за изменений в структуре
   - *Митигация:* Автоматизированная проверка ссылок, регулярный мониторинг

3. **Производительность:** Медленная загрузка из-за множественных сайтов
   - *Митигация:* Оптимизация ресурсов, CDN, мониторинг производительности

4. **SEO проблемы:** Дублирование контента или неправильная индексация
   - *Митигация:* Правильные canonical URLs, уникальные метаданные
