# Этап 4: Развертывание корпоративного сайта на GitHub Pages

## Цель этапа
Запустить корпоративный сайт info-tech.io на GitHub Pages в корневой директории (/) с полным функционалом, автоматическим обновлением контента и производительностью enterprise-уровня.

## Задачи

### 4.1 Подготовка контента корпоративного сайта (1 день)
**Цель:** Финализировать и оптимизировать контент для production развертывания

**Действия:**
- Завершить создание всех разделов корпоративного сайта в info-tech/docs/
- Проверить корректность всех внутренних ссылок и навигации
- Оптимизировать изображения и статические ресурсы
- Добавить SEO метаданные для всех страниц
- Создать sitemap.xml и robots.txt

**Финальная структура контента:**
```
info-tech/docs/content/
├── _index.md                    # Главная: миссия, продукты, CTA
├── about/
│   ├── _index.md               # О проекте и команде
│   ├── mission.md              # Миссия и ценности
│   └── roadmap.md              # Дорожная карта
├── products/
│   ├── _index.md               # Каталог всех продуктов
│   ├── quiz-engine.md          # Quiz Engine + ссылка на /docs/quiz/
│   ├── hugo-templates.md       # Hugo Templates + ссылка на /docs/hugo-templates/
│   ├── web-terminal.md         # Web Terminal + ссылка на /docs/web-terminal/
│   └── info-tech-cli.md        # CLI + ссылка на /docs/info-tech-cli/
├── open-source/
│   ├── _index.md               # Open Source программа
│   ├── contributing.md         # Как стать контрибьютором
│   └── architecture.md         # Техническая архитектура экосистемы
└── blog/
    ├── _index.md               # Технический блог
    └── posts/
        ├── welcome.md          # Приветственная статья
        └── architecture-decisions.md  # Принятые архитектурные решения
```

**Требования к контенту:**
- Все ссылки на документацию ведут на соответствующие разделы /docs/{product}/
- SEO оптимизация: title, description, keywords для каждой страницы
- Адаптивность: корректное отображение на мобильных устройствах
- Производительность: оптимизированные изображения, минимальное время загрузки

### 4.2 Настройка автоматической сборки корпоративного сайта (1 день)
**Цель:** Обеспечить автоматическое обновление корпоративного сайта при изменениях

**Технический workflow:**
```
Изменение в info-tech/docs/content/
    ↓ (GitHub push event)
info-tech/.github/workflows/notify-hub.yml
    ↓ (repository_dispatch event: "corporate-site-update")
info-tech-io.github.io/.github/workflows/deploy-complete-sites.yml
    ↓ (Hugo build с corporate шаблоном)
GitHub Pages: info-tech-io.github.io/ (корневая директория)
```

**Действия:**
- Настроить notify-hub.yml в репозитории info-tech
- Обновить deploy-complete-sites.yml для обработки корпоративного контента
- Протестировать цепочку: изменение в docs/ → автоматический deploy
- Настроить обработку ошибок и уведомления о статусе сборки
- Проверить, что корпоративный сайт размещается в корне (не в /docs/info-tech/)

**Конфигурация сборки:**
- Источник: `info-tech/docs/` → Результат: `info-tech-io.github.io/` (root)
- Шаблон: `corporate` из hugo-templates
- Тема: `compose` (или специализированная корпоративная тема)
- BaseURL: `https://info-tech-io.github.io/`

### 4.3 Интеграция с федерацией документационных сайтов (1 день)
**Цель:** Обеспечить seamless навигацию между корпоративным сайтом и документацией

**Действия:**
- Настроить главное меню с ссылками на все документационные разделы
- Создать единый header/footer для всех сайтов федерации
- Настроить breadcrumbs навигацию
- Добавить поиск по всем сайтам федерации (если технически возможно)

**Навигационная структура:**
```
Главное меню корпоративного сайта:
├── Home (/)
├── Products (/products/)
│   ├── Quiz Engine (/products/quiz-engine/) → ссылка на /docs/quiz/
│   ├── Hugo Templates (/products/hugo-templates/) → ссылка на /docs/hugo-templates/
│   ├── Web Terminal (/products/web-terminal/) → ссылка на /docs/web-terminal/
│   └── CLI Tools (/products/info-tech-cli/) → ссылка на /docs/info-tech-cli/
├── Open Source (/open-source/)
└── Blog (/blog/)
```

**Технические требования:**
- Единый стиль для всех элементов навигации
- Responsive дизайн для мобильных устройств
- Быстрая загрузка (< 2 секунд)
- Доступность (WCAG guidelines)

### 4.4 Тестирование и оптимизация (1 день)
**Цель:** Обеспечить production-ready качество корпоративного сайта

**Функциональное тестирование:**
- Проверка всех страниц на корректность отображения
- Тестирование всех ссылок (внутренние и внешние)
- Проверка навигации и пользовательских путей
- Тестирование на различных устройствах и браузерах

**Производительность:**
- Google PageSpeed Insights Score > 90
- Time to First Byte < 500ms
- Largest Contentful Paint < 2.5s
- Cumulative Layout Shift < 0.1

**SEO проверка:**
- Корректность метаданных на всех страницах
- Наличие структурированных данных (Schema.org)
- Sitemap.xml генерируется автоматически
- robots.txt настроен корректно

**Безопасность и доступность:**
- HTTPS работает корректно
- Нет смешанного контента (mixed content warnings)
- Соответствие WCAG 2.1 AA стандартам
- Корректная работа с отключенным JavaScript

## Риски и митигация

### Возможные риски
1. **Конфликты в навигации:** Сложности с интеграцией корпоративного сайта и документации
   - *Митигация:* Тщательное планирование URL структуры, тестирование навигации

2. **Производительность:** Медленная загрузка из-за большого объема контента
   - *Митигация:* Оптимизация изображений, минификация ресурсов, CDN

3. **SEO проблемы:** Неправильная индексация поисковыми системами
   - *Митигация:* Корректная настройка метаданных, sitemap.xml, тестирование в Search Console
