# Задача 005: Разработка мульти-шаблонной системы сборки hugo-templates

## Общие сведения

**Название задачи:** Разработка мульти-шаблонной системы сборки hugo-templates с поддержкой множественных тем и гибкой конфигурации компонентов

**Дата создания:** 10 сентября 2025  
**Стратегия развертывания:** Blue-Green Deployment (новый репозиторий)  
**Приоритет:** Высокий  
**Длительность:** ~20 рабочих дней (4 недели)

## Ключевая идея

**FROM:** hugo-base (монолитный шаблон) → **TO:** hugo-templates (гибкая система)

### Преимущества Blue-Green подхода:
- ✅ **Zero downtime** - продакшн остается стабильным во время разработки
- ✅ **Безопасная разработка** - возможность экспериментов без риска
- ✅ **Плавная миграция** - постепенный переход модулей по одному
- ✅ **Быстрый откат** - возврат к hugo-base в любой момент

## Архитектурная трансформация

### Монолитная архитектура (hugo-base)
```
hugo-base/
├── hugo.toml           # Единственная конфигурация
├── themes/compose/     # Единственная тема  
├── static/quiz/        # Жестко встроенный Quiz Engine
├── content/            # Фиксированная структура
└── scripts/            # Базовые скрипты
```

### Мульти-шаблонная архитектура (hugo-templates)
```
hugo-templates/
├── templates/          # Множественные шаблоны сборки
│   ├── default/        # Аналог текущего hugo-base
│   ├── minimal/        # Без Quiz Engine  
│   ├── academic/       # Для академических курсов
│   └── enterprise/     # Корпоративное обучение
├── themes/             # Множественные темы (submodules)
│   ├── compose/        # Текущая тема
│   ├── academic/       # Новая академическая тема
│   ├── corporate/      # Корпоративная тема
│   └── minimal/        # Легковесная тема
├── components/         # Модульные компоненты (submodules)  
│   ├── quiz-engine/    # Опциональный Quiz Engine
│   ├── analytics/      # Система аналитики
│   ├── auth/          # Авторизация
│   └── certificates/  # Сертификаты
└── scripts/build.sh   # Параметризованная сборка
```

## Примеры использования новой системы

```bash
# Стандартный модуль (аналог текущего)
./build.sh --template=default --theme=compose --components=quiz-engine

# Быстрая документация без Quiz Engine  
./build.sh --template=minimal --theme=minimal --components=navigation

# Академический курс с расширенной функциональностью
./build.sh --template=academic --theme=academic --components=quiz-engine,references,citations

# Корпоративное обучение с аналитикой
./build.sh --template=enterprise --theme=corporate --components=quiz-engine,analytics,auth,lms-integration
```

## Интеграция в экосистему

### modules.json v2.0 (поддержка dual-repo)
```json
{
  "schema_version": "2.0",
  "modules": {
    "linux_base": {
      "template_repo": "hugo-base",          // ← legacy система
      "build_template": "legacy"            // ← старый режим
    },
    "python_advanced": {
      "template_repo": "hugo-templates",     // ← новая система  
      "build_template": "academic",         // ← академический template
      "theme": "academic",                  // ← академическая тема
      "components": ["quiz-engine", "analytics", "certificates"]
    }
  }
}
```

### CI/CD Pipeline (dual-repo поддержка)
```yaml
env:
  TEMPLATE_REPO: ${{ steps.get-config.outputs.template_repo || 'hugo-base' }}

- name: Checkout template repository
  uses: actions/checkout@v4  
  with:
    repository: info-tech-io/${{ env.TEMPLATE_REPO }}

- name: Build with appropriate system
  run: |
    if [ "${{ env.TEMPLATE_REPO }}" = "hugo-templates" ]; then
      ./build.sh --template="${BUILD_TEMPLATE}" --theme="${THEME}" --components="${COMPONENTS}"
    else
      # Legacy hugo-base система
    fi
```

## План миграции (Blue-Green)

### **Фаза 1: Создание и базовая разработка (5 дней)**
1. **Создание нового репозитория hugo-templates**
2. **Базовая структура** - копирование foundation из hugo-base
3. **Архитектурное планирование** - детальный дизайн системы
4. **Создание templates/** - 4 шаблона сборки
5. **Система themes/** - множественные темы

### **Фаза 2: Функциональная разработка (6 дней)**
6. **Модульные components/** - система компонентов
7. **Параметризованная сборка** - scripts/build.sh
8. **Расширенное тестирование** - все комбинации
9. **Базовая документация** - README и гайды

### **Фаза 3: Интеграция в экосистему (4 дня)**  
10. **Расширение modules.json** - schema v2.0
11. **Обновление CI/CD** - dual-repo поддержка  
12. **Staging тестирование** - полная проверка
13. **Pilot deployment** - 1-2 новых модуля

### **Фаза 4: Production миграция (3 дня)**
14. **Постепенная миграция** - по одному модулю
15. **Production валидация** - мониторинг
16. **Полная документация** - финальные гайды

### **Фаза 5: Завершение (2 дня)**
17. **Финальная миграция** - оставшиеся модули  
18. **Архивирование hugo-base** - deprecated
19. **Очистка CI/CD** - удаление legacy кода
20. **Celebration** - успешная миграция 🎉

## Заключение

Blue-Green подход обеспечивает безопасную трансформацию от монолитной к модульной архитектуре с сохранением стабильности продакшн-системы.
