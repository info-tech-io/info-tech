# Этап 2: Разработка шаблонов сборки (Build Templates)

## Цель этапа
Реализовать 4 шаблона сборки: `default`, `minimal`, `academic`, `enterprise` с различными конфигурациями Hugo и наборами компонентов.

## Длительность
**3 дня** (завершение Фазы 1 + начало Фазы 2)

## Детальные шаги выполнения

### Шаг 2.1: Доработка default template
- Проверить и оптимизировать `templates/default/`
- Убедиться в полной совместимости с текущим hugo-base
- Создать `components.yml` с описанием включенных компонентов
- Добавить метаданные template для системы сборки

### Шаг 2.2: Реализация minimal template
- Создать облегченный `templates/minimal/hugo.toml` без Quiz Engine
- Минимальный набор static файлов
- `components.yml` только с базовой навигацией и контентом
- Быстрая загрузка и минимальные зависимости

### Шаг 2.3: Разработка academic template  
- Расширенная конфигурация Hugo для академических курсов
- Поддержка библиографии, цитирований, references
- `components.yml` с Quiz Engine + References + Citations компонентами  
- **ВАЖНО:** References и Citations компоненты - пока заготовки

### Шаг 2.4: Создание enterprise template
- Корпоративные настройки Hugo с брендингом
- Интеграция с системами аналитики и LMS
- `components.yml` с полным набором: Quiz + Analytics + Auth + LMS Integration
- **ВАЖНО:** Analytics, Auth, LMS Integration - пока заготовки

### Шаг 2.5: Система компонентов components.yml
- Разработать схему описания компонентов в YAML формате
- Создать систему версионирования компонентов
- Реализовать валидацию наборов компонентов
- Документировать какие компоненты реально работают vs примеры

## Способы верификации

### 🔍 Проверка функциональности templates

#### Default Template
```bash
cd templates/default/
hugo config | grep -E "(theme|baseURL|title)"
ls static/quiz/ | wc -l  # Должно быть > 5 файлов
yamllint components.yml
```

#### Minimal Template  
```bash
cd templates/minimal/
hugo config | grep -v quiz  # Не должно содержать quiz настроек
du -sh static/  # Размер должен быть < 50% от default
time hugo --minify  # Время сборки
```

#### Academic Template
```bash
cd templates/academic/
grep -i "academic\|reference\|citation" hugo.toml
yamllint components.yml
grep "status.*planned" components.yml  # Проверить пометки заготовок
```

#### Enterprise Template
```bash
cd templates/enterprise/
grep -i "corporate\|analytics\|auth\|lms" hugo.toml  
yamllint components.yml
grep "status.*planned" components.yml  # Проверить пометки заготовок
```

### 🔍 Проверка совместимости
```bash
# Тест сборки с каждым template
for template in default minimal academic enterprise; do
    echo "Testing $template"
    cd templates/$template/
    hugo --minify --destination ../../test-output/$template/
    [ $? -eq 0 ] && echo "✅ $template build OK" || echo "❌ $template build FAILED"
    cd ../..
done
```

### 🔍 Проверка components.yml
```bash
# Валидация всех components.yml файлов
find templates/ -name "components.yml" -exec yamllint {} \;
find templates/ -name "components.yml" -exec python3 -c "
import yaml, sys
try:
    with open('{}', 'r') as f: 
        data = yaml.safe_load(f)
        print(f'✅ {}: {len(data.get(\"components\", []))} components')
except Exception as e:
    print(f'❌ {}: {e}')
" \;
```

## Риски и митигация

### 🔴 Высокий риск: Совместимость default template
**Риск:** Default template может сломать существующие модули
**Митигация:** Тщательное тестирование со всеми существующими модулями, откат к копии hugo-base при проблемах

### 🔴 Высокий риск: Сложность components.yml схемы
**Риск:** Переусложнение системы компонентов может замедлить разработку
**Митигация:** Начать с простой схемы, итеративно дополнять по потребности

### 🟡 Средний риск: Производительность minimal template
**Риск:** Minimal template может оказаться не намного быстрее default
**Митигация:** Профилирование сборки, агрессивная оптимизация static файлов

### 🟡 Средний риск: Архитектура academic/enterprise заготовок
**Риск:** Неправильная архитектура заготовок усложнит будущую реализацию
**Митигация:** Консультация с архитектурным планом, создание extensible структуры
