# Этап 1: Создание репозитория hugo-templates и базовой архитектуры

## Цель этапа
Создать новый репозиторий `hugo-templates` в организации `info-tech-io` и реализовать базовую файловую структуру мульти-шаблонной системы сборки.

## Длительность
**2 дня** (из 5 дней Фазы 1)

## Детальные шаги выполнения

### Шаг 1.1: Создание репозитория
- Создать новый репозиторий `hugo-templates` в GitHub организации `info-tech-io`
- Настроить базовые настройки репозитория (описание, visibility, branch protection)
- Создать начальную структуру директорий согласно архитектурному плану
- Инициализировать Git с базовыми файлами (README.md, .gitignore)

### Шаг 1.2: Копирование foundation из hugo-base  
- Скопировать рабочие компоненты из существующего `hugo-base`
- Адаптировать структуру под мульти-шаблонную архитектуру
- Сохранить совместимость с текущим функционалом (default template)

### Шаг 1.3: Создание базовой структуры templates/
- Создать директорию `templates/` с поддиректориями для 4 шаблонов
- Реализовать `templates/default/` как копию текущего функционала hugo-base
- Создать заготовки для `templates/minimal/`, `templates/academic/`, `templates/enterprise/`
- **ВАЖНО:** Помечены как примеры - реализация в следующих этапах

### Шаг 1.4: Настройка системы тем themes/
- Создать директорию `themes/` для размещения git submodules
- Добавить текущую тему `compose` как git submodule
- Создать заготовки для планируемых тем (academic, corporate, minimal)
- **ВАЖНО:** Только compose тема реально работает, остальные - примеры

### Шаг 1.5: Базовая документация
- Создать README.md с описанием новой архитектуры
- Задокументировать различия с hugo-base
- Описать статус реализации каждого компонента
- Явно указать что является примерами/заготовками

## Способы верификации

### 🔍 Проверка репозитория
```bash
# Проверить создание репозитория
gh repo view info-tech-io/hugo-templates

# Проверить структуру директорий
ls -la hugo-templates/
tree hugo-templates/ -L 3
```

### 🔍 Проверка git submodules
```bash
# Проверить инициализацию submodule
cd hugo-templates/
git submodule status
git submodule update --init --recursive
```

### 🔍 Проверка default template
```bash
# Проверить функционал default template
cd templates/default/
hugo version
hugo config
ls -la static/ themes/ content/
```

### 🔍 Проверка документации
```bash
# Проверить качество README
wc -l README.md  # Должен быть > 50 строк
grep -i "example\|заготовка\|пример" README.md  # Должен содержать пометки
```

## Риски и митигация

### 🔴 Высокий риск: Git submodules сложность
**Риск:** Проблемы с инициализацией git submodule для compose темы
**Митигация:** Использовать проверенные команды, тестировать на чистом clone

### 🔴 Высокий риск: Потеря функциональности hugo-base
**Риск:** При копировании потерять критически важные компоненты
**Митигация:** Пошаговое копирование с тестированием каждого компонента

### 🟡 Средний риск: Структура директорий
**Риск:** Неправильная структура может усложнить следующие этапы  
**Митигация:** Строгое следование архитектурному плану, проверка tree структуры
